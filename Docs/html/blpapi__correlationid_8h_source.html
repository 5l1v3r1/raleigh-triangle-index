103 class CorrelationId {
104  // A key used to identify individual subscriptions or requests.
105  //
106  // CorrelationId objects are passed to many of the Session object
107  // methods which initiate an asynchronous operations and are
108  // obtained from Message objects which are delivered as a result
109  // of those asynchronous operations.
110  //
111  // When subscribing or requesting information an application has
112  // the choice of providing a CorrelationId they construct
113  // themselves or allowing the session to construct one for
114  // them. If the application supplies a CorrelationId it must not
115  // re-use the value contained in it in another CorrelationId
116  // whilst the original request or subscription is still active.
117  //
118  // It is possible that an application supplied CorrelationId and a
119  // CorrelationId constructed by the API could return the same
120  // result for asInteger(). However, they will not compare equal
121  // using the defined operator== for CorrelationId and there is a
122  // consistent order defined using the defined operator< for
123  // CorrelationId.
124  //
125  // A CorrelationId constructed by an application can contain either
126  //
127  // - a 64 bit integer,
128  //
129  // - a simple pointer or
130  //
131  // - a "smart" pointer object (for example, tr1::shared_ptr)
132  //
133  // For 64 bit integers and simple pointers the values are copied
134  // when CorrelationIds are copied and compared when CorrelationIds
135  // are compared.
136  //
137  // For "smart" pointers the API can accommodate smart pointer classes
138  // that meet the following restrictions.
139  //
140  // - It is no more than sizeof(void*)*4 bytes in size.
141  //
142  // - It is thread safe.
143  //
144  // - It performs all its necessary management as a direct side
145  // effect of their copy constructor and destructor.
146  //
147  // - Its contents are location independent (that is given two
148  // instances of a smart pointer s1 and s2 one can call std::swap(s1,
149  // s2) ).
150  //
151  // The API will embed a smart pointer in the CorrelationId without
152  // allocating memory separately for it. The specified 'smartPtr'
153  // will have its copy constructor invoked when the CorrelationId
154  // is copied and its destructor invoked when the CorrelationId is
155  // destroyed so its resource management will continue to work as
156  // normal.
157  //
158  // CorrelationId's based on a simple pointer and CorrelationId's
159  // based on a smart pointer have the same ValueType
160  // (POINTER_VALUE) which allows them to be compared to each other.
161  //
162  // A CorrelationId based on a simple pointer and a CorrelationId
163  // based on a smart pointer will compare equally with operator==
164  // as long as the pointer is the same.
165  //
166  // Likewise, when comparing two CorrelationId's based on a smart
167  // pointer only the pointer value itself is used for the
168  // comparison, the contents of the smart pointer object are
169  // ignored.

476  TYPE *managedPtr_p = (TYPE*)&managedPtr->userData[0];
