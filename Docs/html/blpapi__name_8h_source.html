89 class Name {
90  // 'Name' represents a string in a form which is efficient for hashing and
91  // comparison, thus providing efficient lookup when used as a key in either
92  // ordered or hash-based containers.
93  //
94  // 'Name' objects are used to identify and access the classes which define
95  // a schema - 'SchemaTypeDefinition', 'SchemaElementDefinition',
96  // 'Constant', and 'ConstantList'. They are also used to access the values
97  // in 'Element' objects and 'Message' objects.
98  //
99  // The 'Name' class is an efficient substitute for a string when used as a
100  // key, providing constant-time hashing and comparision. Two 'Name'
101  // objects constructed from strings for which 'strcmp()' would return 0
102  // will always compare equal.
103  //
104  // The ordering of 'Name' objects (as defined by 'operator<(Name,Name)') is
105  // consistent during a particular instance of a single application.
106  // However, the ordering is not lexical and is not necessarily consistent
107  // with the ordering of the same 'Name' objects in any other process.
108  //
109  // Where possible, 'Name' objects should be initialized once and then
110  // reused. Creating a 'Name' object from a 'const char*' involves a search
111  // in a container requiring multiple string comparison operations.
112  //
113  // Note: Each 'Name' instance refers to an entry in a global static table.
114  // 'Name' instances for identical strings will refer to the same data.
115  // There is no provision for removing entries from the static table so
116  // 'Name' objects should be used only when the set of input strings is
117  // bounded.
118  //
119  // For example, creating a 'Name' for every possible field name and type in
120  // a data model is reasonable (in fact, the API will do this whenever it
121  // receives schema information). Converting sequence numbers from incoming
122  // messages to strings and creating a 'Name' from each one of those
123  // strings, however, will cause the static table to grow in an unbounded
124  // manner, and is tantamount to a memory leak.
